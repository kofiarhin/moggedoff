# Spec Memory

Store generated workflow specs in this folder.

Agents must create a saved spec here before implementation. Use timestamped or slugged filenames, for example:

```txt
_spec/2026-05-10-add-dark-theme.md
```

Each spec should include:

- Request summary.
- Date.
- Source prompt.
- Questions asked and answers received.
- Assumptions.
- Goal.
- Non-goals.
- Users.
- Functional requirements.
- UI expectations, when relevant.
- API expectations, when relevant.
- Data model expectations, when relevant.
- Edge cases.
- Constraints.
- Success criteria.
- Out-of-scope items.
- Open questions.
